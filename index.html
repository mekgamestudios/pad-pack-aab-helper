Overview

PAD Pack AAB Helper is a build-time helper plugin for Unreal Engine Android projects that use Unreal chunking and Google Play Asset Delivery through Android App Bundle builds.

The plugin helps generated non-base chunk pak files become Android asset packs and provides Blueprint access to the generated PAD asset pack names, so developers do not need to manually maintain the asset pack name array.

This plugin does not replace Unreal Engine’s GooglePAD plugin. It assists the AAB asset-pack packaging workflow and provides helper Blueprint functions around generated pack names.

Features

Generated Pack List

Returns the generated PAD asset pack names as a string array for Blueprint workflows.

Automatic AAB Helper

Assists chunk-based pak files being copied/registered as Android asset packs during packaging.

Blueprint Friendly

Provides Blueprint-callable functions for getting the generated asset pack list.

Google PAD Workflow

Designed to work alongside Unreal’s GooglePAD RequestDownload and GetDownloadState nodes.

Installation

  1. Copy the PadPackAABHelper plugin folder into your project’s Plugins directory.
  2. Restart Unreal Editor.
  3. Open Edit → Plugins.
  4. Enable PAD Pack AAB Helper.
  5. Make sure Unreal’s built-in GooglePAD plugin is enabled.
  6. Restart the editor again if prompted.

Project Setup

1. Enable Android App Bundle

Use Android App Bundle packaging for Google Play distribution.

2. Use Chunk Labels

Use Unreal’s chunking system through Primary Asset Labels or your preferred chunk setup. Non-base chunk pak files can be converted into asset packs.

3. Keep Base Content Minimal

Keep startup UI, update/download UI, managers, and required base assets in chunk 0. Gameplay maps and heavy assets should be assigned to non-base chunks.

4. Verify Generated Pak Chunks

pakchunk101-Android_ASTC.pak
pakchunk104-Android_ASTC.pak
pakchunk112-Android_ASTC.pak

Blueprint Usage

Get Generated Asset Pack Names

Use Get Generated PAD Asset Packs to retrieve the generated asset pack names. The returned array can be passed to Unreal’s GooglePAD RequestDownload workflow.

Event Construct / BeginPlay
  → Get Generated PAD Asset Packs
  → Save result to AllPacks array
  → RequestDownload(AllPacks)

Recommended Download Flow

Get Generated PAD Asset Packs
  → RequestDownload
  → Start Timer
  → ForEach AllPacks:
       GetDownloadState
       Switch on DownloadStatus
       Update progress UI
       ReleaseDownloadState
  → If all completed:
       Clear Timer
       Continue to game

Important Notes

  • Do not call Get Generated PAD Asset Packs every timer tick. Cache it once in an array.
  • Always release each download state after reading it.
  • Use GooglePAD’s own RequestDownload, GetDownloadState, and Show Confirmation Dialog nodes for actual PAD download control.

Packaging Workflow

  1. Assign your maps/assets to chunks.
  2. Package the project as Android App Bundle.
  3. The plugin assists the generated pak chunks being included as asset packs.
  4. Upload the AAB to Google Play Console.
  5. Verify asset packs in Play Console if needed.

Troubleshooting

Generated Pack Array Is Empty

  • Make sure non-base pak chunks are actually generated.
  • Check Primary Asset Label chunk IDs.
  • Check whether heavy maps/assets are accidentally referenced by chunk 0.

On-Demand Packs Are Missing

  • Verify staged build paks under Saved/StagedBuilds/Android_ASTC/.../Content/Paks.
  • Check whether the asset was pulled into pakchunk0 by a hard reference.
  • Use UnrealPak -List to inspect pak contents.

Build Fails with Missing Platform Properties

"Modules": [
  {
    "Name": "PadPackAABHelper",
    "Type": "Runtime",
    "LoadingPhase": "Default",
    "PlatformAllowList": [ "Android" ]
  }
]

Download Does Not Continue on Mobile Data

Use GooglePAD’s Show Confirmation Dialog when the download state is REQUIRES_USER_CONFIRMATION.
