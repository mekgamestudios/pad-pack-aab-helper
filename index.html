Usage Workflow (Setup & Asset Pack Generation)

Follow the steps below to correctly configure your project for Google Play Asset Delivery (PAD) and ensure asset packs are generated and detected by the plugin.

Step 1 — Enable Required Plugins

Edit → Plugins

✔ GooglePAD
✔ PAD Pack AAB Helper

Restart Editor

Step 2 — Configure Android Build

Project Settings → Platforms → Android

✔ Build App Bundle (AAB)
✔ For Distribution
❌ Package game data inside APK

Step 3 — Configure Packaging Settings

Project Settings → Packaging

✔ Use Pak File
✔ Generate Chunks
✔ Generate Chunk Install Data
✔ Full Rebuild
✔ For Distribution

❌ Build HTTP Chunk Install Data

Step 4 — Setup Asset Manager (Chunk System)

Project Settings → Asset Manager

Primary Asset Types:
✔ Map
✔ PrimaryAssetLabel

Create Primary Asset Labels:

Content Browser → Right Click → Misc → Primary Asset Label

Settings:
Chunk ID = 101, 102, ...
Cook Rule = Always Cook
Apply Recursively = ON

Step 5 — Assign Assets to Chunks

Chunk 0 → Base (APK)
Chunk 100+ → PAD Asset Packs
If an asset is referenced by Chunk 0, it will NOT be included in PAD. It will be packaged inside the base APK.

Step 6 — Package the Project

File → Package Project → Android (AAB)

Step 7 — Verify Generated Pak Files

Saved/StagedBuilds/.../Content/Paks

Expected output:

pakchunk0-Android_ASTC.pak
pakchunk101-Android_ASTC.pak
pakchunk102-Android_ASTC.pak

Step 8 — Plugin Behavior

✔ Detects generated pakchunkXXX files
✔ Registers them as PAD asset packs
✔ Provides pack list to Blueprint
✔ Ensures inclusion in AAB
The plugin does not modify packaging settings or create chunks. It works on already generated chunk-based pak files.
