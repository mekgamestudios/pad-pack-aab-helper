PAD Pack AAB Helper

Documentation for Unreal Engine plugin used with Google Play Asset Delivery (PAD) and Android App Bundle (AAB).

Overview

This plugin is a helper for Unreal Engine Android projects using chunk-based packaging and Google PAD. It provides generated asset pack names and ensures correct AAB asset pack inclusion.

This plugin does NOT handle downloads. It works together with Unreal's GooglePAD system.

Required Project Setup

1. Enable Google PAD

Edit → Plugins → GooglePAD → Enable

2. Packaging Settings

Project Settings → Packaging

✔ Use Pak File
✔ Generate Chunks
✔ Generate Chunk Install Data
✔ For Distribution
✔ Full Rebuild

❌ Build HTTP Chunk Install Data

3. Asset Manager Setup

Project Settings → Asset Manager

✔ Map
✔ PrimaryAssetLabel

Primary Asset Label:
Chunk ID = 101+
Cook Rule = Always Cook
Apply Recursively = ON

4. Android Settings

Project Settings → Android

✔ Build App Bundle (AAB)
✔ For Distribution
❌ Package game data inside APK

5. Google PAD Settings

Project Settings → Plugins → GooglePAD

✔ Enable Plugin
✔ Only Distribution

Critical Rules

Chunk 0 → Base APK
Chunk 100+ → PAD Asset Packs
If Chunk 0 references an asset, it will NOT be included in PAD. It will be forced into base package.

Build & Verification

File → Package Project → Android (AAB)
Check: 
Saved/StagedBuilds/.../Content/Paks
Expected: 
pakchunk0-Android_ASTC.pak
pakchunk101-Android_ASTC.pak
pakchunk102-Android_ASTC.pak

Plugin Function

✔ Detects generated pakchunkXXX files
✔ Converts them into PAD asset packs
✔ Provides pack list to Blueprint
✔ Ensures AAB inclusion

Limitations

✔ Android only
✔ Requires AAB build
✔ Requires GooglePAD
✔ Requires chunk system

What This Plugin Does NOT Do

❌ Does NOT download asset packs
❌ Does NOT modify ini files
❌ Does NOT create chunks
❌ Does NOT replace GooglePAD system
